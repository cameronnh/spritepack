# Spritepack

Spritepack is a CLI tool that packs individual sprite images into spritesheets and generates Luau data files for use in Roblox projects.

## Installation

Spritepack is installed via [Rokit](https://github.com/rojo-rbx/rokit).

To add spritepack to a new project manually:

```bash
rokit add cameronnh/spritepack
```

## Configuration

Spritepack is configured through a `spritepack.toml` file in the root of your project.

```toml
[paths]
path = "Assets/IndividualSpriteImages"
sheet_output_path = "Assets/Upload/Spritesheets"
data_output_path = "Source/Shared/ImageTools/SpritesheetData"
require_path = "@game/ReplicatedStorage/Source/Shared/Assets"
```

### Fields

| Field | Description |
|---|---|
| `path` | The input directory containing your sprite folders. (e.g. `Assets/IndividualSpriteImages`) holds the individual `.png` images that will be packed into a spritesheet. |
| `sheet_output_path` | The directory where the generated spritesheet `.png` files are saved (e.g. `Items_1.png`, `InventoryIcons_1.png`). These are the packed atlas images ready to be uploaded to Roblox. |
| `data_output_path` | Optional. The directory where the generated `.luau` data files are written. Each file contains a table mapping sprite names to their index and sheet number, along with cell size and column count metadata. |
| `require_path` | Optional. The Luau require alias used in the generated data files to reference your asset module. This is meant to point at the module generated by [Asphalt](https://github.com/jacktabscode/asphalt), which maps asset names to their uploaded Roblox asset IDs. If not specified, the generated data files will just use the spritesheet name directly. |

## Usage

After configuring `spritepack.toml`, run:

```bash
spritepack run
```

Spritepack will scan each subfolder inside your `path` directory, pack the images into spritesheets, and output both the atlas images and corresponding Luau data files to the configured output directories.

## Using with Asphalt

Spritepack pairs well with [Asphalt](https://github.com/jacktabscode/asphalt), a Roblox asset manager. The typical workflow is:

1. Run `spritepack run` to generate your packed spritesheet images. By setting `sheet_output_path` to the same directory that Asphalt watches for uploads (e.g. `Assets/Upload/Spritesheets`), the generated spritesheets are placed directly where Asphalt expects them.
2. Run `asphalt sync` to upload the generated images to Roblox and update your asset ID mappings.

Since the generated Luau data files can reference your asset module via `require_path`, Asphalt keeps those asset IDs up to date so the spritesheets resolve correctly at runtime. This makes the two tools a natural fit — spritepack handles the packing and data generation, and Asphalt handles the uploading and asset management.
